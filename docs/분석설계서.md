## 0. 프로젝트 핵심 정의

서울시 상권분석서비스 데이터를 이용하여 상권·업종·매출·인구 요소와 폐업률의 연관
구조를 확인하고, 유효한 요소를 조합하여 **상권×업종그룹 단위의 다음 분기 폐업률을 회귀
방식으로 예측**한다.

예측 결과는 서울시 및 자치구 정책 담당자가 폐업 위험이 상대적으로 높은 상권·업종 조합을
우선 점검하고, 소상공인 지원 정책의 우선순위를 결정하는 참고 자료로 사용한다.

| 항목 | 정의 |
|---|---|
| 서비스 사용자 | 서울시 및 자치구 소상공인 정책 담당자 |
| 관리 대상 | 상권×업종그룹 조합 |
| 예측 시점 | t분기 데이터가 확정된 시점 |
| 관찰 정보 | t분기까지 확인 가능한 상권·업종·매출·인구 정보 |
| 타깃 | t+1분기 가중 폐업률 |
| 문제 유형 | 회귀 |
| 정책 활용 | 예상 폐업률 기준 우선 점검·지원 대상 선정 |

이 프로젝트에서 `폐업 = 이탈`은 고객 이탈 예측 방법론을 공공정책 영역에 적용하기 위한
비유다. 원천 데이터는 개별 점포 식별자와 점포별 폐업 여부를 제공하지 않으므로, 본
프로젝트는 개별 점포의 폐업 여부나 폐업 확률을 예측하지 않는다. 모델이 예측하는 값은
상권·업종 집계 단위의 다음 분기 폐업률 지표다.

목적은 폐업의 인과관계를 증명하는 것이 아니라, 관찰된 연관 구조를 이용해 다음 분기
폐업률을 예측하고 정책적 점검 우선순위를 제공하는 것이다.

## 1. 분석 구조 (4층 파이프라인)

| 층 | 내용 | 산출물 |
|---|---|---|
| 1층 현황 | 상권별·업종별 분기 폐업률을 있는 그대로 제시 | 분기 시계열 차트, 현황표 |
| 2층 연관 | (A1) 상권 → 매출, (A2) 업종 → 매출, (B) 인구 → 매출 — **각각 독립 분석** | 요소별 분석 결과 |
| 3층 조합 | 요소들을 조합해 확인 (상권×업종 → 매출 등). **2층 무효 요소도 포함해 재검증** | 조합 회귀 + 상관행렬 히트맵, 요소별 최종 판정표 |
| 4층 예측 | t분기 피처로 **t+1분기 폐업률을 회귀 예측** | 저장 회귀 모델, 평가 결과, 예상 폐업률, 위험 순위 |

각 층의 결과가 다음 층의 입력이 된다. 1~3층 담당자는 결과를 "유효/무효 요소 리스트"와 근거 형태로 4층 담당자에게 전달하며, 4층에서는 이를 후보 피처와 해석 근거로 사용한다.

## 2. 데이터 공통 규칙

### 2.1 분석 단위
- 원본 데이터의 기본 행(row) = **세부상권 × 세부업종 × 분기**
- **업종 요인의 분석·더미 변수는 업종그룹 단위로 통일한다 (세부업종 단위 분석 금지, 2026-07-17 팀 결정).** 세부업종(85개)은 원본 식별과 업종그룹 집계에만 사용
- 최종 모델링 단위 = **상권코드 × 업종그룹 × 분기**
- 동일 상권·업종그룹·분기의 세부업종 행은 원건수·금액을 목적에 맞게 먼저 합산한 뒤 비율과 증감률을 다시 계산한다. 하위 행에서 계산한 비율의 평균을 사용하지 않는다.
- 최종 모델링 키 = `상권코드 + 업종그룹 + 기준연분기`
- 집계 계층 = 상권유형(골목/발달/전통시장/관광특구) → 구 → 동 → 세부상권
- 구·동 매핑은 데이터에 이미 포함된 컬럼 사용. 사용 전 아래 3종 검증 필수:
  1. 누락: 구/동이 NaN인 행 0건 확인
  2. 동명이동: 같은 동 이름이 여러 구에 존재하므로 **groupby 키는 반드시 ['구','동'] 쌍** (동 이름 단독 사용 금지)
  3. 행정동/법정동 중 어느 체계인지 확인 후 문서에 기록 (다른 데이터 조인 시 체계 일치 필수)

### 2.2 폐업률 산식 — 가중 집계 (가장 중요한 규칙)
어떤 단위(유형/구/동)로 집계하든 **하위 폐업률의 평균을 내지 말 것.** 반드시 원건수를 합산 후 나눈다.

```
해당 단위 폐업률(%) = Σ(폐업 점포 수) ÷ Σ(전체 점포 수) × 100
```

- 이유: 점포 4개짜리 초소형 상권의 폐업률 50%가 단순 평균을 왜곡함
- 2026-07-17 스팟체크 결과, 원본 폐업률·개업률 컬럼의 분모는 `stores_store_count`가 아니라 **`stores_similar_store_count`**로 확인됐다(유사점포수 기준 일치율 99.93%). 팀 결정에 따라 기존 폐업률·개업률 컬럼은 사용하지 않고, 분모=`stores_store_count`로 원건수에서 직접 가중 집계한다 (`src/closure_rate.py` 공용 함수 사용). 우리 수치가 원본 비율 컬럼과 다른 것은 의도된 차이이며 산출물에 명시한다.
- 본 프로젝트의 폐업률은 개별 점포가 폐업할 확률이 아니라, 특정 상권·업종그룹에서 관찰된 분기 폐업 건수를 해당 단위의 점포 수로 나눈 **집계 지표**다.
- 점포 수는 특정 시점의 스냅샷이고 폐업 수는 분기 중 발생 건수이므로, 일부 관측치에서는 폐업률이 100%를 초과할 수 있다. 예측값을 개인 단위 확률로 해석하거나 근거 없이 0~100 범위로 고정하지 않는다.
- 100% 초과값과 소규모 점포 단위가 모델에 미치는 영향은 Train에서 점검한다. 제외·변환·이상치 완화가 필요하면 기준을 Train에서만 결정하고 Validation/Test에 그대로 적용하며, 처리 전후의 MAE와 위험 순위 성능 및 제외 건수를 함께 기록한다.
- 최소 점포 수 조건은 §5 안건 4에서 결정한다. 적용한다면 배포 시점에 알 수 있는 **t분기 점포 수**를 기준으로 하며, 미래의 t+1분기 점포 수로 표본을 선택하지 않는다.
- 검산 장치: 동 단위 건수를 합치면 구 단위와, 구를 합치면 유형 단위와 정확히 일치해야 함. 안 맞으면 어딘가에서 평균을 낸 것

### 2.3 증감률 산식 — 직전 2분기 평균 대비 (매출·인구 공통)

```
증감률(t) = ( 값(t) − 직전2분기평균 ) ÷ 직전2분기평균 × 100
직전2분기평균 = ( 값(t-1) + 값(t-2) ) ÷ 2
```

- 매출, 유동인구, 상주인구, 직장인구 **모두 이 산식으로 통일** (변수 간 시점 축이 어긋나면 상관분석이 무의미해짐)
- 각 셀(상권×업종그룹)의 최초 2개 분기는 증감률 계산 불가 → 결측 처리
- **계절성 주의:** 이 산식은 계절 효과가 섞임 (예: 4분기는 연말 특수로 대부분 상승처럼 보임). 대응:
  - 모델에 분기 더미 변수(1~4분기 구분) 반드시 추가
  - EDA 해석 시 절대값이 아니라 "같은 분기끼리" 비교

### 2.4 시점 정렬 — 예측의 생명선

**t분기의 피처(매출·인구 등)로 t+1분기의 폐업률을 예측한다.**

- 타깃 `다음분기_폐업률(t+1)`은 해당 상권·업종그룹의 t+1분기 폐업점포수 합계를 점포수 합계로 나누어 계산한다.
- 타깃 생성 순서:
  1. 상권코드×업종그룹×분기 단위로 폐업 수와 점포 수를 합산한다.
  2. 집계된 원건수로 분기 폐업률을 계산한다.
  3. 같은 상권코드×업종그룹 안에서 다음 분기 폐업률을 연결한다.
  4. 현재 분기와 다음 행이 실제 연속 분기인지 검증한다.
  5. 중간 분기가 누락됐거나 다음 분기가 없으면 타깃을 NaN으로 처리한다.
- 단순 `shift(-1)`만 사용하면 중간 분기가 누락된 경우 서로 인접하지 않은 분기가 연결될 수 있으므로, 별도의 분기 연속성 검증을 반드시 수행한다.
- 같은 분기의 피처로 같은 분기의 폐업률을 맞히는 것은 예측이 아님 (이미 일어난 일). 절대 금지
- t+1분기의 매출·인구·점포 정보를 t분기 피처로 사용하지 않는다.
- 마지막 분기는 타깃이 없으므로 학습에서 제외한다.
- 데이터는 다음과 같이 **시간 기준**으로 분리한다. 랜덤 분할은 미래 정보가 섞여 성능이 부풀려지므로 금지한다.

| 구분 | 피처 기간 | 타깃 기간 | 용도 |
|---|---|---|---|
| Train | 2023Q1~2024Q3 | 2023Q2~2024Q4 | 전처리기와 모델 학습 |
| Validation | 2024Q4~2025Q2 | 2025Q1~2025Q3 | 모델·하이퍼파라미터 선택 |
| Test | 2025Q3 | 2025Q4 | 최종 성능 평가 |

- 2021~2022년은 현황 분석에만 사용하고 모델 학습·선택·평가에서 제외한다.
- 모든 전처리기는 Train 데이터에만 `fit`한다.
- Validation으로 모델과 하이퍼파라미터를 결정하고 Test는 최종 결정 뒤 한 번만 평가한다.

### 2.5 데이터 제외 규칙 (팀 결정 2026-07-17)

- **업종그룹 '법무·회계·부동산·전문서비스'는 전 층(1~4층) 분석에서 제외한다.**
  - 근거: 실제 구성 업종이 '통번역서비스' 1개뿐으로 그룹명이 내용을 대표하지 못하며(법무·회계·부동산 업종은 데이터에 없음), 매출 기록도 전무해 2층 이후 분석이 불가능
- **세부업종 '핸드폰'은 전 층 분석에서 제외한다.**
  - 근거: 통신사·대리점 정책에 따라 같은 자리에서 폐업↔다른 사업자 개업을 수시로 반복하는 업종이라, 폐업 건수가 실제 영업 중단 위험을 반영하지 못함 (매출 증감률 하위 1위로 나타나는 것도 이 왜곡의 영향 가능)
- 구현: `src/data_rules.py`의 `apply_exclusions()`를 모든 분석 스크립트에서 데이터 로드 직후 호출 (industry_group·industry_name 컬럼 필수)
- 이 제외는 산출물·발표에 명시한다 (§4 규칙 4)

## 3. 층별 상세 스펙

### 1층. 현황 제시 (담당: EDA)
- 상권유형별 분기 폐업률 시계열, 업종별 분기 폐업률 시계열 (가중 집계)
- 구·동 단위 현황표 (지도 시각화는 시간 남으면)
- **연도별 카테고리별(상권유형·업종그룹) 폐업률 추이 1장을 최우선으로 산출** → 코로나 구간 처리 방침 결정에 사용 (§5 미결 안건 참조). 단일 전체 평균이 아니라 카테고리별로 제시 (2026-07-17 팀 확인)
- 참고 수치: 서울시 전체 분기 개업률 약 1.9%, 폐업률 약 2.7% (개업 11,714 / 폐업 16,916건 기준) — 순감소 국면이라는 문제 정의의 근거

### 2층-A1. 상권 → 매출 (담당: 통계분석)
- **상권(유형) 단독으로** 매출 증감률과의 연관을 확인 (업종과 섞지 않음)
- 방법: 상권유형별 매출 증감률 분포 비교 (그룹 간 차이 검정 또는 상권유형 더미만 넣은 단순 회귀)
- 종속변수: 매출 증감률 (§2.3 산식). 매출 "금액"이 아니라 "증감률"을 쓰는 이유: 금액은 규모 효과(강남이라 큼)만 잡히고 상승/하락 방향이 묻힘

### 2층-A2. 업종 → 매출 (담당: 통계분석)
- **업종 단독으로** 매출 증감률과의 연관을 확인 (상권과 섞지 않음)
- 방법: 업종별 매출 증감률 분포 비교 (A1과 동일 방식, 업종 기준)
- A1·A2는 각각 독립적으로 수행하고 결과를 분리해 기록 — 상권×업종을 함께 보는 것은 3층의 일

### 2층-B. 인구 → 매출 (담당: 통계분석)
- 유동·상주·직장인구 **증감률** ↔ 매출 **증감률**의 피어슨 상관 + 산점도
- **수준(명 수)끼리 상관 금지** — "큰 상권은 인구도 매출도 많다"는 당연한 규모 상관(0.9대)이 나와 무의미함
- 세 인구 유형은 반드시 분리 분석 (오피스가는 직장인구, 관광지는 유동인구가 결정적일 수 있음 — 이 차이 자체가 발견)

### 3층. 조합 확인 (담당: 통계분석 + EDA)
- 2층에서 각각 확인한 요소들을 **조합**해서 확인하는 단계: 상권×업종 → 매출, (유효했다면) 인구까지 포함한 조합
- **중요: 2층에서 단독으로 유의하지 않았던 요소도 3층 조합에 반드시 포함해 재검증한다.** 단독으로는 안 보이던 연관이 조합에서 드러나는 경우가 있음 (예: 상주인구가 오피스 상권과 주거지 상권에서 반대로 작용해 전체에서는 상쇄되어 보이는 경우 → 상권유형과 결합하면 유의해짐). 2층의 무효 판정은 "탈락"이 아니라 "단독 무효"라는 기록일 뿐
- 방법: 회귀분석에 상권유형·업종 더미를 함께 투입 + 교호작용 항으로 "상권×업종 조합 효과" 확인. 인구 변수도 전부 포함
- 상관행렬 히트맵: 전체 1장 + **상권유형별로 쪼갠 버전** (전체에선 안 보이는 상관이 특정 유형 안에서만 강한 패턴 = 심슨의 역설 포착)
- 유동인구↔매출은 강한 다중공선성 예상 → 4층에서 두 변수의 중요도가 서로 나눠 먹혀 작아 보일 수 있음을 해석에 반영
- 산출물: 요소별 최종 판정표 — ① 단독 유의 ② 단독 무효·조합 유의 ③ 조합에서도 무효, 세 분류로 기록 (③도 버리지 말 것 — "확인했으나 유의하지 않았다"는 성실성의 근거)

### 4층. 다음 분기 폐업률 예측 (담당: 모델링)

#### 4.1 예측 방식

- 다음 분기 폐업률을 **연속값으로 예측하는 회귀 방식**을 사용한다.
- 예시: "영등포구청역 2번 상권의 일반의류 업종그룹은 다음 분기 폐업률이 12.4%로 예측되며, 전체 관리 대상 중 예상 위험 상위 7%에 해당한다."
- 이진 분류를 사용하지 않는 이유:
  - 원천 데이터에는 개별 점포별 폐업 여부 0/1 타깃이 없다.
  - 폐업률은 연속형 집계 지표다.
  - 임의의 고위험 cutoff를 타깃으로 사용하면 기준에 따라 정답이 달라진다.
  - 연속 예측값을 유지하면 정책 상황에 따라 우선 관리 범위를 조정할 수 있다.

#### 4.2 비교 모델

동일한 시간 분할과 동일한 피처를 사용해 다음 모델을 비교한다.

1. `DummyRegressor`: 평균 또는 중앙값을 예측하는 최소 기준 모델
2. `LinearRegression` 또는 `Ridge`: 회귀계수 방향과 크기를 확인하는 해석 기준 모델
3. `RandomForestRegressor`: 비선형 관계와 변수 간 조합 효과 비교
4. `XGBRegressor`, `LightGBMRegressor` 또는 `CatBoostRegressor`: 최종 예측 성능 후보

모델 종류와 하이퍼파라미터는 Train과 Validation 결과로 선택한다. Test 결과를 확인한 뒤
모델이나 파라미터를 변경하지 않는다.

#### 4.3 입력 피처

- 후보 피처: 매출 증감률, 유동·상주·직장인구 증감률, 점포당 매출, 현재 점포 수, 상권유형, 업종그룹, 분기 더미, 3층에서 확인한 교호작용·조합 피처
- 3층의 통계적 유의성만으로 피처를 기계적으로 제거하지 않는다. 3층 결과는 후보 피처와 해석 근거로 사용하고, 최종 포함 여부는 Validation 성능·누수 가능성·재현성·해석 가능성을 함께 고려한다.

#### 4.4 평가 지표

- **MAE(주 지표):** 평균적으로 몇 %p의 폐업률 오차가 발생하는지 평가
- **RMSE:** 큰 예측 오차에 더 큰 페널티를 부여
- **R²:** 기준 변동 대비 모델 설명력 확인
- 폐업률 0인 관측치가 많을 수 있으므로 MAPE는 주 평가 지표로 사용하지 않는다.
- 정책 활용 성능:
  - 예측 폐업률 순위와 실제 폐업률 순위의 Spearman 상관계수
  - 예측 상위 K%의 실제 평균 폐업률
  - 예측 상위 K%와 실제 상위 K%의 중첩 비율
  - 전체 평균 대비 예측 상위 K%의 실제 폐업률 배수
- 영향도는 회귀계수(방향·해석)와 피처 중요도(기여 크기)를 병행 제시하되 인과효과로 해석하지 않는다.

#### 4.5 고위험군 후처리

- 고위험 여부는 모델 학습에 사용하는 0/1 타깃이 아니다. 모델은 모든 유효 관측치의 다음 분기 폐업률을 연속값으로 학습한다.
- 고위험군은 회귀 예측이 완료된 뒤 정책 활용을 위해 구분한다.

| 위험등급 | 예시 기준 |
|---|---|
| 고위험 | 예상 폐업률 순위 상위 10% |
| 주의 | 예상 폐업률 순위 상위 10~30% |
| 일반 | 나머지 70% |

- 상위 비율은 예시이며 서울시가 한 분기에 점검하거나 지원할 수 있는 관리 대상 수를 고려해 결정한다.
- 고위험 기준을 데이터로 조정한다면 Validation에서만 결정하고 Test에는 고정하여 적용한다.
- 높은 실제 폐업률을 보인 행만 추출하여 회귀모델을 학습해서는 안 된다.
- 실제 고폐업률 사례는 문제 심각성을 보여주는 EDA·발표 사례, 위험 순위 평가, 큰 오차 사례 분석에만 사용한다.

#### 4.6 모델 저장과 추론

- 최종 전처리기와 회귀모델을 하나의 Pipeline으로 저장한다.
- 모델과 함께 피처 이름·순서·자료형, 학습 기간·데이터 버전, 타깃 산식, 평가 지표, 고위험 후처리 기준, Python·주요 패키지 버전을 기록한다.
- 저장 모델을 새 프로세스에서 다시 불러와 동일 입력에 대해 예측 가능한지 검증한다.
- Streamlit에서는 모델을 재학습하지 않고 저장된 Pipeline의 `predict()`를 호출한다.

#### 4.7 Streamlit 결과 제공

Streamlit은 서울시 및 자치구 정책 담당자를 사용자로 가정한다.

1. **서울시 폐업 현황:** 서울시 전체 및 자치구·상권유형·업종그룹별 폐업률 추이, 최근 분기 고폐업 현황, 제외 업종·기간 안내
2. **모델 성능:** 모델별 MAE·RMSE·R², 실제값과 예측값 비교, 잔차 분포, 피처 중요도·회귀계수, 예측 상위 K% 포착 성능
3. **다음 분기 위험 예측:** 상권과 업종그룹을 선택하면 예상 폐업률, 전체 예측 분포 내 백분위, 위험등급, 최근 폐업률 비교, 주요 연관 피처, 정책 점검 우선순위를 제공

출력 예시:

> 다음 분기 예상 폐업률: 12.4%<br>
> 서울시 전체 예측 분포 중 상위 7%<br>
> 위험등급: 고위험<br>
> 활용 제안: 현장 점검 및 지원 필요성 우선 검토

화면에는 예측값이 개별 점포의 폐업 확률이 아니며, 실제 정책 집행 전 추가 현장 확인이
필요하다는 안내를 표시한다.

```python
@st.cache_resource
def load_model():
    return joblib.load("models/closure_rate_pipeline.joblib")

model = load_model()
input_df = pd.DataFrame([form_data])
predicted_closure_rate = model.predict(input_df)[0]
```

## 4. 표현·해석 공통 규칙 (기획서/발표 전체 적용)

1. **"영향을 준다" → "유의한 연관 / 선행 신호"로 통일.** 관측 데이터의 상관은 인과 입증이 아님. "근본 원인 규명" 표현도 금지 → "위험 신호 식별"
2. 폐업률·증감률 수치엔 항상 **"분기 기준"** 명시 (연간 수치와 혼동 방지)
3. 매출은 명목 금액 기준 → 물가 보정 안 함을 한계로 각주 명시
4. 모든 가정·제외 규칙(코로나 처리 등)은 숨기지 말고 문서에 명시. 가정은 틀려도 되지만 숨기면 안 됨
5. 모델 출력은 개별 점포 폐업 확률이 아니라 **상권×업종그룹 단위 폐업률 예측값**이라고 표현한다.
6. `고객 이탈`은 프로젝트 방법론의 비유로만 사용하고, 서비스 설명에서는 `상권·업종 단위 폐업 위험 예측`을 우선 사용한다.
7. 고위험등급은 회귀 예측값의 정책적 후처리이며 모델의 타깃이 아니다.
8. 제외 기간·제외 업종·이상치·최소 점포 수 조건을 산출물에 명시한다.

## 부록. 자주 하는 실수 체크리스트

- [ ] 폐업률을 평균 냈다 → ❌ 건수 합산 후 나누기
- [ ] 같은 분기 피처로 같은 분기 폐업률을 예측했다 → ❌ shift(-1)로 다음 분기 타깃
- [ ] `shift(-1)`만 사용해 누락 분기를 건너뛰었다 → ❌ 실제 인접 분기 검증
- [ ] train/validation/test를 랜덤 분할했다 → ❌ 시간 기준 분리
- [ ] 인구 "명 수"와 매출 "금액"의 상관을 봤다 → ❌ 증감률끼리
- [ ] 동 이름만으로 groupby 했다 → ❌ ['구','동'] 쌍
- [ ] 4분기 증감률이 높다고 "상승세"로 해석했다 → ❌ 계절 효과 확인 (분기 더미 / 같은 분기끼리 비교)
- [ ] 오버샘플링·스케일링을 테스트셋에 적용했다 → ❌ 학습셋 기준으로만 fit
- [ ] 개별 점포 폐업 확률이라고 표현했다 → ❌ 집계 단위 폐업률 예측값
- [ ] 고폐업률 사례만 추출해 회귀모델을 학습했다 → ❌ 모든 유효 연속형 타깃 사용
- [ ] 고위험등급을 모델의 0/1 타깃으로 설명했다 → ❌ 회귀 예측 후 정책적 후처리
- [ ] 통계적 유의성만으로 피처를 제거했다 → ❌ Validation 성능·누수·재현성 함께 검토
- [ ] 저장 모델을 앱에서 다시 학습했다 → ❌ 저장 Pipeline을 로드해 `predict()` 호출
- [ ] Test 결과를 본 뒤 모델·파라미터·고위험 기준을 바꿨다 → ❌ Validation에서 결정 후 Test 1회 평가
